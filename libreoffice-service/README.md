# SlideForge – LibreOffice conversion service

Microservice that converts a PDF to a real, vector-based PPTX using
LibreOffice's headless engine. Each shape, text run and image in the PDF
becomes a native PowerPoint object, fully editable in PowerPoint, Keynote
or Google Slides.

This runs as a Docker container; it can NOT run on Vercel (Vercel functions
don't support shipping the LibreOffice binary). Deploy it once on a free-tier
container host and point the SlideForge web app at its URL.

## Endpoint

`POST /convert`
- `Content-Type: multipart/form-data`
- field `file`: the PDF (max 25 MB)
- optional header `Authorization: Bearer <SHARED_SECRET>`
- response: `application/vnd.openxmlformats-officedocument.presentationml.presentation`

`GET /healthz` → `{ ok: true, ... }`

## Local test

```bash
cd libreoffice-service
docker build -t slideforge-lo .
docker run -p 8080:8080 -e SHARED_SECRET=devsecret slideforge-lo
# in another shell:
curl -X POST http://localhost:8080/convert \
  -H "Authorization: Bearer devsecret" \
  -F "file=@/path/to/slide.pdf" \
  -o out.pptx
```

## Deploy options (all free)

### Fly.io (recommended — fast cold starts, generous free tier)

```bash
cd libreoffice-service
flyctl launch --no-deploy --name slideforge-lo
flyctl secrets set SHARED_SECRET=$(openssl rand -hex 32)
flyctl deploy
```

Note: `fly.toml` will be generated by `launch`. After deploy, copy the URL
(e.g. `https://slideforge-lo.fly.dev`) and the secret into Vercel env:
- `LIBREOFFICE_URL=https://slideforge-lo.fly.dev`
- `LIBREOFFICE_SECRET=<the secret>`

### Render.com (free tier, sleeps after 15 min idle)

1. Push this repo to GitHub.
2. On Render: New → Web Service → connect repo → root `libreoffice-service`.
3. Runtime: Docker. Plan: Free.
4. Add env var `SHARED_SECRET` (any random string).
5. Deploy. Copy the URL into Vercel env as above.

### Hugging Face Spaces (free, Docker SDK)

1. Create a new Space → SDK: Docker.
2. Upload these three files (`Dockerfile`, `package.json`, `server.js`) and a
   minimal `README.md` with `sdk: docker` frontmatter.
3. Set the secret `SHARED_SECRET` in Space settings.
4. Use `https://<user>-<space>.hf.space` as the LIBREOFFICE_URL.

## SlideForge integration

The Next.js app calls this service from `/api/pdf-to-pptx`, proxied so the
secret never reaches the browser. The "Conversione vettoriale (LibreOffice)"
button in the editor uses this path.

If `LIBREOFFICE_URL` is unset on Vercel, the button is hidden and the app
falls back to its existing raster-based export.
